# Performance log

Each entry follows the same shape:

1. **Query.** The SQL we are tuning, with bound parameters described.
2. **Workload.** Row counts, distribution, what we ran to populate.
3. **Baseline.** `EXPLAIN (ANALYZE, BUFFERS, FORMAT TEXT)` before any change.
4. **Change.** The index, schema, or rewrite we made — and why.
5. **After.** `EXPLAIN (ANALYZE, BUFFERS)` after.
6. **Commentary.** Two or three sentences on what we learned.

## Template — copy this for each case study

### Query

```sql
SELECT id, event_type, properties
FROM events
WHERE user_id = $1
  AND created_at >= $2
  AND created_at <  $3
ORDER BY created_at DESC
LIMIT 100;
```

`$1` is a uniformly random user_id; `$2`/`$3` is a 24h window over the last 7 days.
